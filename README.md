# Desafio_DOM
Visualização e Gerenciamento de Usuários na Tabela

## *Nome:* Visualização e Gerenciamento de Usuários na Tabela

## Descrição:
````
Como um administrador do sistema, desejo criar uma tabela que exiba uma lista dos usuários
presentes em nossa base de dados. Essa tabela deve incluir um recurso de pesquisa para facilitar a
localização de usuários específicos, além de oferecer a funcionalidade de edição e exclusão
de registros de usuário.
````
##  Critérios de Aceitação:
````
Como administrador, desejo acessar uma página ou interface dedicada à visualização de usuários.
  
Na página de visualização de usuários, deve haver uma tabela que liste todos os usuários da nossa base de dados.
  
  A tabela deve exibir as seguintes informações de cada usuário: nome, cep, endereço de e-mail,
data de cadastro e quaisquer outros campos relevantes.
  
Deve haver um campo de pesquisa que permita ao administrador buscar usuários com base em critérios como nome, sobrenome ou e-mail.
  
Quando um termo de pesquisa for inserido, a tabela deve ser atualizada automaticamente para
exibir apenas os resultados correspondentes ao critério de pesquisa.
  
Ao lado de cada registro de usuário na tabela, deve haver opções para editar e excluir o usuário.
  
A opção de "Edição" deve levar o administrador a uma página onde ele pode modificar as informações do usuário,
como nome, sobrenome, e-mail, etc.
  
A opção de "Exclusão" deve solicitar uma confirmação antes de remover permanentemente o usuário do sistema.
  
Após editar ou excluir um usuário, a tabela deve ser atualizada automaticamente para refletir as mudanças feitas.
````
## Definições de Pronto:
````
A tabela de visualização de usuários deve estar funcional e exibir informações corretas.
A funcionalidade de pesquisa deve retornar resultados precisos.
As operações de edição e exclusão de usuários devem ser bem-sucedidas sem erros.
A interface deve ser intuitiva e de fácil utilização.
Testes de unidade e integração devem ser concluídos com sucesso.
O código deve ser revisado e documentado conforme necessário.
O usuário não deve encontrar erros ou problemas significativos ao utilizar essa funcionalidade.
````
